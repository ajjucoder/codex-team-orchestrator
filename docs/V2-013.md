# V2-013 Hierarchical Orchestration Tools

## Scope
Add parent/child orchestration tools for child-team lifecycle, scoped delegation, and hierarchy rollups with strict subtree isolation.

## Implemented
1. New hierarchical tools:
   - `mcp/server/tools/hierarchy.ts`
   - `team_child_start`
   - `team_child_list`
   - `team_delegate_task`
   - `team_hierarchy_rollup`
2. Tool contracts and schemas:
   - `mcp/schemas/tools/team_child_start.schema.json`
   - `mcp/schemas/tools/team_child_list.schema.json`
   - `mcp/schemas/tools/team_delegate_task.schema.json`
   - `mcp/schemas/tools/team_hierarchy_rollup.schema.json`
   - `mcp/schemas/contracts.ts`
3. Validation tests:
   - `tests/unit/v2-013.hierarchy-tools.test.ts`
   - `tests/integration/v2-013.hierarchy-tools.integration.test.ts`

## Acceptance Criteria
1. Parent teams can create and monitor child teams.
2. Delegated tasks can only target a descendant team of the parent.
3. Cross-tenant/cross-tree delegation attempts are denied.
4. Hierarchy rollups return deterministic aggregate visibility.

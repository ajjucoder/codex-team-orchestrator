# V2-003 Runtime Permission Enforcement Middleware

## Scope
Enforce per-agent runtime permission profiles on every tool invocation and support per-tool/per-action access checks.

## Implemented
1. Runtime permission decision engine:
   - `mcp/server/permission-profiles.ts`
2. Server middleware enforcement with explicit deny reasons:
   - `mcp/server/server.ts`
3. Unit + integration deny/allow matrix coverage:
   - `tests/unit/v2-003.permission-enforcement.test.ts`
   - `tests/integration/v2-003.permission-enforcement.integration.test.ts`

## Acceptance Criteria
1. Unauthorized tool calls fail with explicit permission-denied reasons.
2. Authorized tool calls keep existing behavior.
3. Per-action rules can override per-tool defaults.

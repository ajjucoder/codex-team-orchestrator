# V2-005 Team Operating Mode Data Model

## Scope
Introduce persisted team runtime mode state (`default`, `delegate`, `plan`) with backward-compatible migration defaults.

1. Team mode persistence migration:
   - `mcp/store/migrations/004_team_mode.sql`
2. Store/entity support for mode read/write:
   - `mcp/store/entities.ts`
   - `mcp/store/sqlite-store.ts`
3. Team lifecycle status API now reports mode:
   - `mcp/server/tools/team-lifecycle.ts`
4. Contract updates and migration/status tests:
   - `mcp/schemas/entities/team.schema.json`
   - `mcp/schemas/contracts.ts`
   - `tests/unit/v2-005.team-mode.test.ts`
   - `tests/integration/v2-005.team-mode.integration.test.ts`

## Acceptance Criteria
1. Existing teams migrate to `mode=default`.
2. New teams default to `mode=default`.
3. Mode state persists and is visible via status APIs.
